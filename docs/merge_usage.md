# Tamatr - Tool for Aggregating & Merging ATaRVa TR_vcfs

<p align=center>
  <img src="../lib/tmtr_logo.png" alt="Logo of Mirchi" width="200"/>
</p>

Tamatr is a tool for merging VCF files generated by ATaRVa, accessible via the command `atarva merge`. It is implemented using the `pysam` and `polars` packages and supports multi-threaded execution for improved performance.

The tool allows merging only specific contigs of interest across VCF files using the `--contigs` option followed by one or more chromosome names (e.g., `--contigs chr5 chr7 chr8`). Additionally, when the input VCFs contain more loci than required, only the regions specified in the BED file are merged across samples.

The `-i` / `--vcfs` option accepts multiple VCF files as input. Alternatively, a text file containing the paths to all VCF files can also be supplied.

## Usage
```
usage: atarva merge [-h] -r <FILE> -i <FILE> [<FILE> ...] -f <FILE> [--contigs CONTIGS [CONTIGS ...]] [-o <STR>] [-p <INT>]

Required arguments:
  -r <FILE>, --regions <FILE>
                        input regions file. the regions file should be strictly in bgzipped tabix format. If the regions input file is in bed format. First sort it using bedtools. Compress it using
                        bgzip. Index the bgzipped file with tabix command from samtools package.
  -i <FILE> [<FILE> ...], --vcfs <FILE> [<FILE> ...]
                        text file containing paths to input vcf files to be merged. The text file should list each path on a separate line. The vcf files should be strictly in bgzipped tabix format. If
                        the vcfs input file is in vcf format. First sort it using bcftools. Compress it using bgzip. Index the bgzipped file with tabix command from samtools package.
  -f <FILE>, --fasta <FILE>
                        input reference fasta file. The file should be indexed.

Optional arguments:
  --contigs CONTIGS [CONTIGS ...]
                        contigs to get merged [chr1 chr12 chr22 ..]. If not mentioned every contigs in the region file will be merged.
  -o <STR>, --outname <STR>
                        name of the output file, output is in vcf format.
  -t <INT>, --threads <INT>
                        number of threads. [default: 1]
```

## Examples
### Basic usage
```bash
atarva merge -i file1.vcf.gz file2.vcf.gz file3.vcf.gz -r TR_regions.bed.gz -f reference.fasta
```

### With vcf text file
```bash
atarva merge -i vcf_files.txt -r TR_regions.bed.gz -f reference.fasta
```

### With multithreading
```bash
atarva merge -i file1.vcf.gz file2.vcf.gz file3.vcf.gz -r TR_regions.bed.gz -f reference.fasta -t 4
```
In all the above examples, the output of Tamatr is saved to atarva_merged.vcf unless `-o or --outname` is specified.
